# Yandex Calendar Reminder для macOS

Нативное macOS-приложение для напоминаний о встречах из Яндекс Календаря с полноэкранными уведомлениями.

## Возможности

- Синхронизация с Яндекс Календарём через CalDAV
- Настраиваемые интервалы напоминаний (по умолчанию: за 20 мин и 2 мин)
- Полноэкранные уведомления, чтобы точно не пропустить встречу
- Ручное добавление событий
- Иконка в менюбаре с просмотром предстоящих встреч
- Системные уведомления macOS
- Звуковые оповещения

## Требования

- macOS 13.0 (Ventura) или новее
- Xcode 15+

## Быстрый старт

### 1. Откройте проект

```bash
cd YandexCalendarReminder
open Package.swift
```

Xcode откроет проект как Swift Package. Нажмите **Cmd+R** для запуска.

### 2. Настройте Яндекс аккаунт

1. Создайте **пароль приложения** в Яндексе:
   - Перейдите на [id.yandex.ru](https://id.yandex.ru)
   - Безопасность → Пароли приложений → Создать пароль
   - Выберите тип: "Почта" или "Календарь"
2. В приложении откройте **Настройки** (иконка шестерёнки в менюбаре)
3. Вкладка «Аккаунт» → введите логин и пароль приложения
4. Нажмите «Проверить подключение»

### 3. Настройте напоминания

- Вкладка «Напоминания» → добавьте/удалите интервалы
- Включите/выключите полноэкранные и звуковые уведомления

## Добавление событий вручную

Нажмите кнопку **«+»** в менюбаре → заполните название, время и место встречи.

## Структура проекта

```
YandexCalendarReminder/
├── Package.swift
└── YandexCalendarReminder/
    ├── App.swift                    # Точка входа
    ├── AppDelegate.swift            # Полноэкранные уведомления
    ├── Info.plist
    ├── Models/
    │   ├── CalendarEvent.swift      # Модель события
    │   └── ReminderSettings.swift   # Настройки
    ├── Services/
    │   ├── YandexCalDAVService.swift # Синхронизация с Яндексом
    │   └── ReminderService.swift    # Управление напоминаниями
    └── Views/
        ├── MenuBarView.swift        # Главное окно менюбара
        ├── SettingsView.swift       # Настройки
        └── AddEventView.swift       # Добавление событий
```

## Безопасность

- Пароль приложения хранится в UserDefaults (для продакшена рекомендуется Keychain)
- Используется HTTPS для всех запросов к CalDAV
- Основной пароль Яндекса **не используется** — только пароль приложения
